1 Prerequisites

The installer is fully self-contained — it bundles the .NET 8 runtime so there is nothing to pre-install on the target server. Just verify the following:

RequirementDetails
Operating SystemWindows Server 2019, 2022, or Windows 10/11 (x64)
PowerShellVersion 5.1+ (built into Windows)
PermissionsAdministrator (Run as Administrator)
Disk Space~500 MB for application + database growth
NetworkOutbound HTTPS to graph.microsoft.com
ⓘ
No .NET SDK or runtime installation is needed. The publish output includes everything required.

2 Build the Release Package

On your development machine (where the source code is), open PowerShell and run the build script:

# Navigate to the deploy directory
cd deploy\windows-service

# Build a self-contained win-x64 release
.\Build-Release.ps1PowerShell

This runs dotnet publish with the following settings:

SettingValue
ConfigurationRelease
Runtimewin-x64 (self-contained)
Outputdeploy\windows-service\publish\

When complete, the publish folder contains everything needed — the application binaries, the .NET runtime, and the installer scripts.

Custom output path

.\Build-Release.ps1 -OutputPath "C:\builds\cloudativ-v1.0"PowerShell

3 Transfer to the Target Server

Copy the entire publish folder to the target Windows Server using any method:

MethodExample
File shareCopy to \\server\c$\temp\cloudativ\
RDPDrag & drop into a Remote Desktop session
USBCopy to a USB drive and plug into the server
SCP / SFTPscp -r publish/ admin@server:C:\temp\cloudativ\
ⓘ
The folder can be placed anywhere temporarily. The installer will copy the files to C:\Program Files\Cloudativ Assessment\ during installation.

4 Run the Installer

4.1 — Open PowerShell as Administrator

Right-click the Start menu and select "Windows PowerShell (Admin)" or "Terminal (Admin)".

4.2 — Navigate to the publish folder

cd "C:\temp\cloudativ"  # wherever you copied the filesPowerShell

4.3 — Run the installer

.\Install-CloudativAssessment.ps1PowerShell

The installer runs 8 automated steps:

StepActionWhat Happens
[1/8]Check existingDetects prior install, stops the service if upgrading
[2/8]Create directoriesCreates Program Files and ProgramData directories
[3/8]Copy filesSyncs application binaries using robocopy /MIR
[4/8]HTTPS certificateGenerates a self-signed cert (RSA 2048, 5-year validity)
[5/8]ConfigurationWrites appsettings.Production.json with paths and cert
[6/8]Windows ServiceRegisters service with auto-start and auto-recovery
[7/8]FirewallOpens inbound TCP port 5443 (Domain & Private profiles)
[8/8]Start & verifyStarts service, polls /health endpoint

5.3 — Open in browser

Navigate to https://YOUR-SERVER:5443 in any browser. You will see a certificate warning for the self-signed cert — click Advanced > Proceed to continue.

ⓘ
Replace the self-signed certificate with a real one from your CA for production use. Place the .pfx file in C:\ProgramData\Cloudativ Assessment\certs\ and update the path and password in appsettings.Production.json.

7 Custom Install Options

The installer accepts parameters to customize the deployment:

ParameterDefaultDescription
-InstallPathC:\Program Files\Cloudativ AssessmentApplication binary directory
-DataPathC:\ProgramData\Cloudativ AssessmentDatabase, logs, and certificate directory
-HttpsPort5443HTTPS port number
-CertificateSubjectMachine hostnameCN for the self-signed certificate
-ServiceNameCloudativAssessmentWindows Service name
-ServiceDisplayNameCloudativ AssessmentDisplay name in services.msc
-SkipFirewall(off)Skip firewall rule creation
-Force(off)Skip upgrade confirmation prompt

Examples

# Custom HTTPS port
.\Install-CloudativAssessment.ps1 -HttpsPort 8443

# Install to a different drive
.\Install-CloudativAssessment.ps1 -InstallPath "D:\Apps\Cloudativ" -DataPath "D:\Data\Cloudativ"

# Use a specific hostname for the certificate
.\Install-CloudativAssessment.ps1 -CertificateSubject "assessment.company.com"

# Silent install (no prompts)
.\Install-CloudativAssessment.ps1 -Force

# Skip firewall (when behind a reverse proxy or external firewall)
.\Install-CloudativAssessment.ps1 -SkipFirewallPowerShell

8 Directory Layout

The installer separates binaries from data so that upgrades never touch your database, logs, or certificates.

Auto-recovery

The service is configured to restart automatically on failure:

FailureActionDelay
1st failureRestart10 seconds
2nd failureRestart10 seconds
SubsequentRestart30 seconds

The failure counter resets after 24 hours of uptime.

10 Upgrading

To upgrade to a new version, simply re-run the installer from the new build output. The installer is idempotent — it will:

  1. Stop the existing service
  2. Sync updated binaries (using robocopy /MIR)
  3. Preserve the existing HTTPS certificate
  4. Update the service binary path
  5. Restart the service
# Build the new version
.\Build-Release.ps1

# Copy publish folder to the server, then run:
.\Install-CloudativAssessment.ps1 -ForcePowerShell
✓
Your database, logs, and certificates in C:\ProgramData\ are never touched during upgrades. EF Core migrations run automatically on startup.

11 Uninstalling

Remove service and binaries (keep data)

cd "C:\Program Files\Cloudativ Assessment"
.\Uninstall-CloudativAssessment.ps1PowerShell

This stops and removes the Windows Service, deletes the firewall rule, and removes the application binaries. Your database, logs, and certificates are preserved.

Remove everything (including data)

.\Uninstall-CloudativAssessment.ps1 -RemoveData

WARNING: This will PERMANENTLY delete:
  - Database:     C:\ProgramData\Cloudativ Assessment\data
  - Logs:         C:\ProgramData\Cloudativ Assessment\logs
  - Certificates: C:\ProgramData\Cloudativ Assessment\certs

Type 'DELETE' to confirm data removal: DELETEPowerShell
⚠
Data removal is irreversible. Back up your database before running with -RemoveData.

12 Troubleshooting

Service won't start

# Check Windows Event Log for errors
Get-EventLog -LogName Application -Source "CloudativAssessment" -Newest 10

# Check application logs
Get-Content "C:\ProgramData\Cloudativ Assessment\logs\cloudativ-*.log" -Tail 100PowerShell

Port already in use

# Find what's using port 5443
netstat -ano | findstr :5443

# Reinstall on a different port
.\Install-CloudativAssessment.ps1 -HttpsPort 8443 -ForcePowerShell

Certificate errors

# Delete the existing cert and re-run installer to regenerate
Remove-Item "C:\ProgramData\Cloudativ Assessment\certs\cloudativ-assessment.pfx"
.\Install-CloudativAssessment.ps1 -ForcePowerShell

Database locked / migration error

# Stop the service, back up the database, and restart
Stop-Service CloudativAssessment
Copy-Item "C:\ProgramData\Cloudativ Assessment\data\cloudativ_assessment.db" ".\backup.db"
Start-Service CloudativAssessmentPowerShell

Firewall blocking access from other machines

# Verify the firewall rule exists
Get-NetFirewallRule -DisplayName "Cloudativ Assessment*" | Format-List

# If missing, re-create it
New-NetFirewallRule -DisplayName "Cloudativ Assessment (HTTPS 5443)" `
    -Direction Inbound -Protocol TCP -LocalPort 5443 -Action AllowPowerShell

Reset admin password

If you've lost access, delete the database and restart the service. A fresh database will be created with the default admin credentials.

Stop-Service CloudativAssessment
Remove-Item "C:\ProgramData\Cloudativ Assessment\data\cloudativ_assessment.db"
Start-Service CloudativAssessment
# Default login restored: admin@cloudativ.local / Admin@123!PowerShell
⚠
Deleting the database removes all data including tenants, assessments, and users. Only do this as a last resort.
